## 第零章、需求验证框架

用"场景 + 困惑 + 初步想法"来检验这个方案是否清晰：

### 核心场景 + 困惑 + 初步想法

**【场景】**  
用户处于分手期间（分手发生、分手进行中或摇摆不决中），整天反复纠缠于"要不要挽回"、"我是不是做错了"、"要不要主动找TA"等念头，最难熬的时刻想要冲动地联系TA。

**【困惑】**  
- 竞品AI（听海、分了么等）都是泛化套话，看不到"我"的具体故事细节
- 用户需要的不是被分类，而是**被理解**：AI要记住我和TA的故事，在我冲动时提醒我
- 不同的人在分手的不同阶段，需要的陪伴方式完全不同

**【初步想法】**  文档
如果产品能：
1. 通过对话式故事录入，记住用户"和TA的完整故事"
2. 根据用户当前所处的阶段（分手后/分手中/摇摆期），**用不同的AI人设和功能侧重**来陪伴
3. AI在每次回应时都**引用用户的故事细节**，而不是套话

那么用户就会感觉被"真正理解"而不是被"机器对待"。

### 检验结果

✅ **一句话能否说清:**  
"分手期间走不出来的人群，需要一个记住他们故事、根据阶段而定制陪伴的AI"  
→ 清楚了

✅ **目标人群是否单一:**  
不是单一，而是**一个人群的三个不同阶段**（分手后→分手中→摇摆期），都属于"分手中走不出来"  
→ 符合，用1套产品通过场景分流解决

✅ **核心差异化是否明确:**  
"对话式故事录入 + 场景分流AI"  
→ 明确了

## 二、场景设计

### 一个入口，三条路径

不做三个产品，不做三个页签。**同一套UI，后台自动分流**。

故事录入第一问选择场景 → 存入数据库 `scene` 字段 → AI 的 system prompt 根据字段切换。

```
"你现在和TA的关系是？"
  A. 已经分手了（分手后）
  B. 我们在谈/刚谈分手（分手进行中）
  C. 我自己一直在想要不要分（摇摆期）
```

| 场景 | AI定位 | 功能侧重 |
|------|--------|----------|
| 分手后 | "陪你走出来" | 打卡戒断 + 清醒备忘 |
| 分手中 | "帮你看清这段关系" | AI深度对话 + 关系复盘 |
| 摇摆期 | "帮你想清楚再决定" | 理性分析 + 利弊梳理 |

**实现成本：零额外开发量。** 只是AI prompt多一行场景标识。

## 三、核心差异化：对话式故事录入

### 为什么不做测试题

| 竞品做法（测试题） | 失恋晴天做法（对话式故事录入） |
|-------|---------|
| 给用户贴标签："你是回避型依恋" | 记住用户的故事："你说他不回消息那天最难受" |
| 用户感觉：被分类 | 用户感觉：**被理解** |
| AI没有上下文 | AI始终有故事档案 |

### 故事录入流程（聊天气泡形式）

| 序号 | 问题 | 输入方式 | 目的 |
|------|------|----------|------|
| Q1 | 你现在和TA的关系是？ | 3个选项 | 场景分流 |
| Q2 | 你们在一起多久了？ | 时间选项 | 关系基础信息 |
| Q3 | 是谁先开口说分手的？ | 3个选项（我提/TA提/协议） | 关系动态 |
| Q4 | 大概是因为什么？ | **开放输入** | 核心故事素材 |
| Q5 | 最让你反复回想的是哪件事？ | **开放输入** | 情感锚点 |
| Q6 | 你现在最常冒出来的念头是？ | 多选 | 当前状态 |

前3问用选项降低门槛，后3问引导开放输入获取真实故事。

### 故事档案摘要

录入完成后，AI生成一段摘要展示给用户：

> "你们在一起3年，是TA提的分手。你说最难受的是那天发消息TA一直没回……你现在一方面心疼自己，一方面又在想是不是自己哪里不够好。"

**让用户第一次就感受到：这个AI懂我。**

## 六、AI Prompt 设计

### 6.1 System Prompt 结构

```
你是"失恋晴天"——用户的清醒好友。你了解她的完整故事：

【用户故事档案】
- 场景：[分手后/分手中/摇摆期]
- 关系时长：{duration}
- 分手发起方：{initiator}
- 分手原因：{reason}（用户原话）
- 最难熬的事：{hardest_moment}（用户原话）
- 当前主要念头：{current_thoughts}

【回应原则】
1. 永远先认情绪，再说其他
2. 每次回应必须引用用户故事中的至少1个具体细节
3. 绝对不说泛化套话："这很正常"、"给自己时间"、"你值得更好的"
4. 根据用户触发的模式切换语气：
   - 陪伴模式（"我只是想说说话"）：温柔倾听，不建议，先共情
   - 清醒模式（"我想联系TA"）：理性，帮用户回想分手原因，指出情绪规律
   - 复盘模式（"帮我分析关系"）：结构化提问，帮用户梳理关系模式
5. 每次最多问1个问题
6. 语气：像懂你的朋友，不是心理咨询师，不是机器人
7. 回复控制在100字以内，简短有力
```

### 6.2 AI回应对比示例

**竞品听海的回应：**
```
用户："我今天又想联系他了"
AI："我理解你的感受，分手确实很痛苦。给自己一些时间，
    你值得被更好地对待。试着做一些让自己开心的事吧。"
```

**失恋晴天的回应：**
```
用户："我今天又想联系他了"
AI："是那种突然冒出来的念头吗？你之前说最难受的是
    发消息他一直不回那天——你现在想联系他，是想要
    一个回应，还是只是习惯了有他在？"
```

## 七、视觉风格

## 七前、情绪支持与陪伴策略（MVP 极简版）

核心只做一件事：在最难熬、最想联系TA的时刻，给出基于用户故事的清醒提醒，让他/她度过戒断期。后续才扩展到复盘/成长。

- 触发：用户主动点“我想联系TA”或文本含“想联系/忍不住/很想他/她”。
- 话术：先复述故事细节（分手原因/最难熬那天），再问一个澄清：“你此刻是想得到回应，还是怕自己错过？”
- 输出：1 条清醒提醒 + 1 个30秒替代动作（呼吸/走路/写不联系理由）；字数<80字。
- 安全：强烈负面情绪词时跳过广告一次，并提示线下求助渠道。
- 衡量：一周内自报“想联系”频次下降；连续3天未联系。

## 八、广告变现策略

### 8.1 开通条件

- 累计 **500 UV** 即可开通流量主（个人主体可申请）
- 后台路径：微信公众平台 → 推广 → 流量主
- 结算周期：每月1号、15号，腾讯代扣个税，7个工作日到账

### 8.2 广告位设计

| 优先级 | 类型 | 位置 | 单次收益 | 说明 |
|--------|------|------|----------|------|
| 最高 | **封面广告** | 打开小程序时全屏 | ~¥0.5 | 占总收入50%+，**必须手动开启** |
| 高 | **激励视频** | AI对话前 | ~¥0.2 | 每日首次免费，后续看广告解锁 |
| 中 | **插屏广告** | 页面切换时 | 较低 | 补充收入 |

**不做的广告：**
- ❌ Banner（收益极低）
- ❌ 打卡完成后（伤留存）
- ❌ 清醒备忘内（伤体验）

### 8.3 广告接入方式

- **MVP用智能接入**：后台扫码→手机拖拽→确认，不写代码
- 后期精细化时再切自主开发接入

### 8.4 收入预估

| 广告类型 | 计算 | 日收入 |
|----------|------|--------|
| 封面广告 | DAU 1000 × ¥0.5 | ¥500 |
| 激励广告 | DAU 1000 × 50%转化 × ¥0.2 | ¥100 |
| **合计** | | **~¥600/天（理想状态）** |

初期DAU小，先不追求收入最大化，优先保留存。

### 8.5 后期升级路径

```
流量主验证需求（0-3个月）
    ↓
注册个体工商户 + 对公账户
    ↓
开通微信虚拟支付
    ↓
```
